Warning:
Code should be considered to be of "Alpha" quality!

Future versions will definitely break the current API. I hope to freeze the API for version 0.5.0, have all JUnits in place by version 0.7.5, with minor API improvements (but 100% reverse compatible with 0.5.0). The final "BETA" series will start when version 0.9.0 is released. All targeted platforms will have hopefully been realized, and releases will only contain bug fixes from that point on.

Commons-SSL will mostly stop development once 1.0.0 is hit. The problem we're solving with Commons-SSL is quite small, so I don't see any reason to ever go beyond 1.0.0, except for fixing bugs.
